# Glossary

Kurze Erklaerungen fuer zentrale Begriffe, Tools und Plattformen im Projektkontext.

## OpenRouter

Eine Plattform, ueber die verschiedene KI-Modelle ueber eine einheitliche API angesprochen werden koennen.
Im Projektkontext ist das vor allem dann relevant, wenn unterschiedliche Modelle fuer Coding, Recherche oder Textarbeit verglichen werden.

## Git

Ein Versionskontrollsystem fuer Dateien und Code.
Mit Git koennen Aenderungen nachvollzogen, Branches erstellt und Zusammenarbeit ueber Pull Requests organisiert werden.

## GitHub

Eine Plattform fuer Repositories, Issues, Pull Requests und Projektorganisation.
In diesem Projekt ist GitHub die zentrale Quelle fuer Aufgaben und organisatorische Dokumentation.

## OpenCode

Ein KI-gestuetzter Coding-Agent bzw. Workflow-Ansatz fuer Entwicklungsarbeit.
Im Projektkontext ist damit ein Tool gemeint, das beim Lesen, Schreiben und Strukturieren von Code oder Repo-Inhalten unterstuetzt.

## Zed

Ein moderner Code-Editor mit Fokus auf Geschwindigkeit und kollaborative Entwicklung.
Kann fuer das Projekt als Editor fuer Code, Markdown und organisatorische Dateien genutzt werden.

## WebXR

Browser-Schnittstelle fuer Virtual und Augmented Reality.
Im Produktkontext von `Neural Flight` die Grundlage dafuer, VR direkt im Browser auf der Quest auszufuehren.

## ICAROS

Ein Fitness- und Bewegungsgeraet, auf dem Nutzerinnen und Nutzer durch Koerperverlagerung Flugbewegungen steuern.
Im Projekt dient es als physische Eingabe fuer Pitch und Roll.
