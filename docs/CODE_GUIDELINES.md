# Code Guidelines

Gemeinsame Richtlinien fuer Repo-Arbeit, Git-Workflow, Doku und Zusammenarbeit.

## Grundsaetze

- Kleine, nachvollziehbare Aenderungen statt grosser unklarer PRs
- Lesbarkeit vor Cleverness
- Konkrete Benennungen fuer Dateien, Seiten und Eintraege
- Dokumentation aktualisieren, wenn sich Arbeitsweise oder Regeln aendern
- Dieses Repo bleibt organisatorisch, nicht produktorientiert

## GitHub-Workflow

- Jede groessere Arbeit basiert auf einem Issue
- Pull Requests verlinken das passende Issue
- Unklare Entscheidungen werden vor dem Mergen dokumentiert oder geklaert

## Code-Stil

- Bestehende Konventionen dieses Repos respektieren
- Keine unnoetigen Format- oder Strukturwechsel in fremden Dateien
- Markdown klar, knapp und gut scanbar halten
- Neue Dateien und Ordner klar und konsistent benennen

## Review

- Fokus auf Verstaendlichkeit, Struktur und Wartbarkeit
- Risiken und offene Fragen klar benennen
- Bei Studi-Beitraegen lieber kleine Verbesserungszyklen als ueberladene Einmal-PRs

## Dokumentation

- Neue Begriffe ins Glossar aufnehmen, wenn sie haeufig im Projekt auftauchen
- Relevante Links in [`docs/LINKS.md`](LINKS.md) sammeln
- Tutorial-Ideen in [`docs/TUTORIAL_IDEAS.md`](TUTORIAL_IDEAS.md) pflegen
